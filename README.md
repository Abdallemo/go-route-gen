# go-route-gen

End-to-end type-safe routing toolkit for Go 1.22+ and TypeScript.

This toolkit generates TypeScript definitions directly from Go standard library (`net/http`) handlers. It provides strict, tRPC-style type safety across the network boundary without requiring heavy backend frameworks.

The toolkit consists of two parts:

1. **CLI**: A Go AST scraper that extracts your routes.
2. **Client**: A strictly-typed Axios wrapper (NPM package) that consumes the extracted routes.

## Installation

### 1. Install the CLI

```bash
go install github.com/Abdallemo/go-route-gen/cmd/route-gen@latest

```

### 2. Install the Frontend Client

```bash
npm install @abdallemo/routegen-client axios

```

## CLI Usage

The CLI parses your Go files, finds `mux.Handle` or `http.HandleFunc` declarations, and extracts the `METHOD /path` strings introduced in Go 1.22.

### Configuration

Create a `routegen.json` file in your Go project root to set your default paths:

```json
{
  "handlers_dir": "./cmd/api/handlers",
  "out_path": "../frontend/src/lib/route.ts"
}

```

### Commands and Flags

Run the CLI to generate the TypeScript routes file.

```bash
route-gen

```

**Available Flags:**

* `-w, --watch`: Watch the handlers directory for file changes and regenerate automatically.
* `-d, --dir`: Path to the backend handlers directory (overrides `routegen.json`).
* `-o, --out`: Path to the output TypeScript file (overrides `routegen.json`).

Example:

```bash
route-gen --watch --dir ./internal/handlers --out ./web/src/routes.ts

```

## Client Usage

The NPM package consumes the generated TypeScript file to provide strict autocomplete and parameter validation in your frontend application.

### Initialization

Instantiate the client using the generated route dictionary.

```typescript
// src/lib/api.ts
import { GoApiClient } from '@abdallemo/routegen-client'
import { API_ROUTES } from './route' // Generated by the Go CLI

export const api = new GoApiClient<typeof API_ROUTES>({
  baseURL: 'http://localhost:8080',
  // axiosInstance: customAxios, // Optionally pass a configured Axios instance
})

```

### Making Requests

The client enforces exact string matches for routes. If a route contains dynamic segments (e.g., `{id}`), TypeScript will force you to provide an object with those exact keys.

```typescript
// Static route: No parameters expected
const userRes = await api.expect<User>().request('GET /users/me')

// Dynamic route: Requires the matching parameter object
const deleteRes = await api.expect<SuccessResponse>().request(
  'DELETE /customer/{customerId}', 
  { customerId: 1,  } // TS compiler error if omitted or misspelled
)

```

### Strict Mode

By default, the client only accepts route strings that exist in the generated `route.ts` file.

To bypass this restriction (e.g., to hit external APIs) while maintaining autocomplete for your internal routes, pass `false` as the second generic parameter during initialization:

```typescript
export const apiLoose = new GoApiClient<typeof API_ROUTES, false>({
  baseURL: 'http://localhost:8080'
})

// Autocomplete still works for Go routes, but arbitrary strings are now permitted
await apiLoose.request('GET /external-service/webhook')

```
